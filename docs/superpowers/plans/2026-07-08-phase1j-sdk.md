# Phase 1j: SDK (`@cadence/sdk`) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build `@cadence/sdk`, a thin, typed, isomorphic (Node + browser) TypeScript client wrapping every REST endpoint of `apps/api` that is actually callable with an API key.

**Architecture:** One internal `request()` helper (native `fetch`, JSON in/out, throws a typed `CadenceError` on any non-2xx response) sits behind a top-level `Cadence` class. Each REST resource area (plans, subscriptions, customers, invoices, analytics, webhook-endpoints, webhook-deliveries, merchants) gets its own small class constructed with a reference to `request()`, assembled onto the `Cadence` instance. A pure, network-free `verifySignature` helper reverses the HMAC scheme `apps/worker` already signs webhook deliveries with.

**Tech Stack:** TypeScript 5.7, native `fetch` (no HTTP runtime dependency), Vitest with mocked `globalThis.fetch` (no real API server needed), CommonJS build output (`tsc -p tsconfig.build.json`) matching every other package in this monorepo.

## Global Constraints

- API-key auth only (`Authorization: Bearer <apiKey>`). No session-cookie support in this phase.
- `cadence.apiKeys.*` and `cadence.merchants.create()` are NOT built — both routes are `@UseGuards(SessionGuard)`-gated with no API-key-authenticated path (confirmed via direct inspection of `apps/api/src/api-keys/api-keys.controller.ts` and `apps/api/src/merchants/merchants.controller.ts`).
- No on-chain helpers (`cadence.onchain.*`), no `cadence.payouts`, no `cadence.events` — none of `/v1/prepare/*`, `/v1/payouts`, `/v1/events` exist as real backend endpoints yet.
- Constructor is `new Cadence({ apiKey: string, baseUrl?: string })`. `baseUrl` defaults to `"http://localhost:3000"` (this project has no hosted API deployment; matches `apps/api`'s own local dev port).
- List methods return the raw pagination envelope `{ data: T[], has_more: boolean, next_cursor: string | null }`, typed generically. No auto-pagination iterator.
- Any non-2xx HTTP response is parsed from the standard `{ error: { type, code, message, param? } }` envelope (exact shape defined in `apps/api/src/common/errors.ts`) and thrown as a `CadenceError extends Error` carrying `{ type, code, message, param?, status }`.
- Build output is CommonJS (`module: "CommonJS"`, `moduleResolution: "Node10"`), matching `packages/db`'s and `packages/shared`'s `tsconfig.build.json` pattern exactly — do not introduce ESM output as a monorepo outlier.
- `packages/sdk` has zero runtime dependencies. `@types/node`, `typescript`, `vitest` as devDependencies only (matching `packages/shared`'s devDependency list).
- Money amounts in all API responses are raw on-chain-unit strings (e.g. `amount: "20000000"`), never pre-divided by the SDK — the SDK passes these through verbatim as `string`, matching what the API actually returns (grepped: no endpoint returns a separate `amount_usd` field despite the PRD's §7.4 conventions section mentioning one — analytics endpoints return pre-formatted USD strings like `mrr_usd: "1000.000000"` instead; do not invent a normalization the API doesn't provide).

export type CadenceErrorType = "authentication_error" | "permission_error" | "invalid_request_error" | "rate_limit_error" | "api_error";

export class CadenceError extends Error {
  readonly type: CadenceErrorType;
  readonly code: string;
  readonly param?: string;
  readonly status: number;

  constructor(params: { type: CadenceErrorType; code: string; message: string; param?: string; status: number }) {
    super(params.message);
    this.name = "CadenceError";
    this.type = params.type;
    this.code = params.code;
    this.param = params.param;
    this.status = params.status;
  }
}

- [ ] **Step 7: Implement `request()`**

Create `packages/sdk/src/request.ts`:

```typescript
import { CadenceError, type CadenceErrorType } from "./errors.js";

export type RequestFn = (
  method: string,
  path: string,
  options?: { query?: Record<string, string | undefined>; body?: unknown },
) => Promise<unknown>;

interface ErrorEnvelope {
  error: { type: CadenceErrorType; code: string; message: string; param?: string };
}

function isErrorEnvelope(value: unknown): value is ErrorEnvelope {
  return (
    typeof value === "object" &&
    value !== null &&
    "error" in value &&
    typeof (value as ErrorEnvelope).error === "object" &&
    typeof (value as ErrorEnvelope).error.type === "string" &&
    typeof (value as ErrorEnvelope).error.code === "string"
  );
}

function buildUrl(baseUrl: string, path: string, query?: Record<string, string | undefined>): string {
  const url = new URL(path, baseUrl);
  if (query) {
    for (const [key, value] of Object.entries(query)) {
      if (value !== undefined) url.searchParams.set(key, value);
    }
  }
  return url.toString();
}

export function createRequest(apiKey: string, baseUrl: string): RequestFn {
  return async (method, path, options) => {
    const url = buildUrl(baseUrl, path, options?.query);
    const headers: Record<string, string> = { Authorization: `Bearer ${apiKey}` };
    let body: string | undefined;
    if (options?.body !== undefined) {
      headers["Content-Type"] = "application/json";
      body = JSON.stringify(options.body);
    }

    const response = await fetch(url, { method, headers, body });
    const text = await response.text();
    const parsed: unknown = text.length > 0 ? JSON.parse(text) : undefined;

    if (!response.ok) {
      if (isErrorEnvelope(parsed)) {
        throw new CadenceError({ ...parsed.error, status: response.status });
      }
      throw new CadenceError({ type: "api_error", code: "unknown_error", message: `Request failed with status ${response.status}`, status: response.status });
    }

    return parsed;
  };
}
```

Note: `buildUrl` uses `new URL(path, baseUrl)`, so `path` must always start with `/` (e.g. `/v1/plans`) — every resource class in later tasks must call `request()` with a leading-slash path, matching every example in this task's own test.

import type { RequestFn } from "../request.js";
import type { PageEnvelope, Plan } from "../types.js";

export interface ListPlansFilter {
  active?: boolean;
  limit?: number;
  startingAfter?: string;
}

export interface AttachPlanMetadataBody {
  name: string;
  description?: string;
  imageUrl?: string;
  dunningLadder?: string[];
}

export class PlansResource {
  constructor(private readonly request: RequestFn) {}

  async list(filter: ListPlansFilter = {}): Promise<PageEnvelope<Plan>> {
    const query: Record<string, string | undefined> = {};
    if (filter.active !== undefined) query.active = String(filter.active);
    if (filter.limit !== undefined) query.limit = String(filter.limit);
    if (filter.startingAfter !== undefined) query.starting_after = filter.startingAfter;
    return this.request("GET", "/v1/plans", { query }) as Promise<PageEnvelope<Plan>>;
  }

  async get(onchainId: string): Promise<Plan> {
    return this.request("GET", `/v1/plans/${onchainId}`) as Promise<Plan>;
  }

  async attachMetadata(onchainId: string, body: AttachPlanMetadataBody): Promise<Plan> {
    return this.request("POST", `/v1/plans/${onchainId}/metadata`, { body }) as Promise<Plan>;
  }
}

```typescript
import { createHmac, timingSafeEqual } from "node:crypto";

export function verifySignature(rawBody: string, header: string, secret: string): boolean {
  const parts = header.split(",").reduce<Record<string, string>>((acc, part) => {
    const [key, value] = part.split("=");
    if (key && value) acc[key] = value;
    return acc;
  }, {});

  const timestamp = parts["t"];
  const providedSig = parts["v1"];
  if (!timestamp || !providedSig) return false;

  const expectedSig = createHmac("sha256", secret).update(`${timestamp}.${rawBody}`).digest("hex");

  const expectedBuf = Buffer.from(expectedSig, "hex");
  const providedBuf = Buffer.from(providedSig, "hex");
  if (expectedBuf.length !== providedBuf.length) return false;

  return timingSafeEqual(expectedBuf, providedBuf);
}
```

Note the `expectedBuf.length !== providedBuf.length` guard: `timingSafeEqual` throws (rather than returning `false`) if given two buffers of different lengths, so a malformed or truncated `v1` value must be checked before calling it — this is exactly what the fourth test ("rejects a malformed header") exercises indirectly via a header with no `v1=` segment at all (caught earlier by the `!providedSig` check), and what a genuinely-wrong-length hex string would exercise if it reached this far.
